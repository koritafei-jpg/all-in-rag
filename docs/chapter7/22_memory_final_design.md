# Memory 系统最终设计方案（终稿）
## —— 自进化认知记忆架构

## 1. 文档目标

设计一套可落地、高性价比、兼具前沿性的 AI Memory 系统。方案在工程上采用“三层记忆 + 混合检索 + 轻量内存图 + 离线巩固 + 主动遗忘 + 自进化控制 + 安全防护”的组合架构，保证系统可用、可演进、可治理。

目标：

1. 记忆按工作记忆、情景记忆、事实记忆三层管理，并引入完整生命周期：抽取、巩固、遗忘、冲突消解、审计回滚。  
2. 检索采用 BM25 + 向量 + 时间感知 + 图补充的多路融合，支持字段过滤、MMR 重排、动态配额。  
3. 不依赖持久化图数据库，采用关系型数据库 + 向量数据库 + 轻量内存图的混合存储。  
4. 引入可控自进化：检索权重、遗忘参数、图结构、抽取策略可在评测与灰度中持续优化。  
5. 分阶段落地：先稳定可用，再增强智能，最终实现可治理的 Agentic Memory。  

---

## 2. 设计理念与创新定位

### 2.1 开源框架能力边界与本方案应对

- **Mem0**：生产化 memory layer 能力强，但复杂图推理与主动遗忘需业务补齐。  
  - 本方案：轻量内存图 + 遗忘机制 + 证据链约束。  
- **Zep/Graphiti**：时间图能力突出，但图引擎体系运维复杂。  
  - 本方案：图索引内存化、可重建，降低基础设施复杂度。  
- **Letta/MemGPT**：Agent 自主记忆能力强，但自编辑需治理。  
  - 本方案：提议-审核模式，禁止高风险直写事实源。  
- **LightMem/SCM**：睡眠巩固、遗忘机制启发明显。  
  - 本方案：离线巩固 + 主动遗忘 + 成本治理。  

### 2.2 论文引用映射与收益

- 分层记忆（MemGPT/Letta + Tulving）：降低上下文压力，提升跨会话稳定性。  
- 记忆中间层（Mem0）：工程接入快，便于多业务复用。  
- 时间图检索（Zep/Graphiti、AriGraph）：支持当前事实/历史事实并存，减少时序错配。  
- 睡眠巩固与遗忘（LightMem、SCM、Ebbinghaus）：降低冗余，提升记忆新鲜度与成本效率。  
- Agentic 提议（AgeMem/Memoria）：增强记忆更新灵活性，同时保持审核可控。  
- 混合检索（BM25/MMR/RAG）：覆盖率、精准度、多样性兼顾。  
- 安全治理（AgentWard）：降低投毒与越权风险，支持审计与回滚。

## 3. 总体架构（五平面协同）

1. **在线写入平面**：消息标准化、抽取、冲突消解、写入索引、图增量更新。  
2. **在线检索平面**：多路召回、融合打分、重排、动态配额返回。  
3. **离线认知平面**：睡眠巩固、主动遗忘、画像更新、审计巡检。  
4. **自进化控制平面**：反馈收集、策略搜索、沙箱评估、灰度发布、择优应用。  
5. **安全防护平面**：输入清洗、投毒检测、决策对齐、高风险执行拦截。  

存储原则：
- **关系库 + 向量库是事实源**  
- **轻量内存图是可重建索引层，不作为单点事实源**

## 4. 三层记忆模型与数据定义

### 4.1 三层定义

- **工作记忆（WM）**：上下文窗口内短期状态，不持久化。  
- **情景记忆（Episodic）**：带时间、来源、会话语义的事件记录。  
- **事实记忆（Semantic）**：可复用、可版本化的稳定事实与偏好。  

### 4.2 关键字段要求（摘要）

**EpisodeMemory**
- 必备：`id/user_id/session_id/event_time/text/is_valid/version/source`
- 扩展：`entities/relations/consolidation_status/risk_flags`

**SemanticMemory**
- 必备：`id/user_id/text/version/is_valid/episode_memory_ids`
- 治理：`confidence/provenance_hash/reinforce_count/pinned`

关键约束：
- `episode_memory_ids` 必填（事实必须可追溯证据）  
- 低 `confidence` 事实默认不进入高优先检索  
- 写入必须可审计（trace_id）

## 5. 写入流程（Ingest）

### 5.1 在线流程
用户/Agent输入 → 安全校验 → 情景抽取 → 情景写入与索引 →（条件触发）事实抽取与冲突消解 → 版本写入 → 图更新。

### 5.2 工程必备约束
- **幂等键**：`request_id + event_id`（唯一键建议含 `user_id`）  
- **一致性**：事实写入采用版本乐观锁，不允许静默覆盖  
- **补偿机制**：图更新失败入队异步重建  
- **可追踪性**：全链路 `trace_id` 联动日志  

### 5.3 性能目标（SLO）
- 检索服务：P95 < 300ms（不含 LLM 生成），P99 < 600ms  
- 写入链路：P95 < 500ms  
- 可用性：月度 >= 99.9%

## 6. 检索流程（混合召回 + 时间感知）

### 6.1 融合打分
\[
FinalScore_i = w_v \cdot VecSim_i + w_k \cdot BM25_i + w_t \cdot TimeDecay_i + w_{imp} \cdot Imp_i + w_g \cdot GraphScore_i
\]

建议：
- `GraphScore` 先归一化到 [0,1]，避免单通道主导  
- 初始权重：`w_v=0.45, w_k=0.30, w_t=0.10, w_imp=0.05, w_g=0.10`

### 6.2 记忆强度更新（防无界增长）
\[
S_i \leftarrow clamp(S_{min}, S_{max}, (1-\gamma)S_i + \lambda_{pos} \cdot pos\_fb - \lambda_{neg} \cdot neg\_fb)
\]
\[
TimeDecay_i = exp(-\Delta t / S_i)
\]

### 6.3 重排与过滤
- MMR 去重增强多样性  
- 阈值过滤（默认 0.55）  
- 权限过滤（user/agent/scope/is_valid）  
- 风险过滤（risk_flags 降权或人工审核）

### 6.4 降级策略
- L0：BM25+Vector+Graph（全量）  
- L1：Graph降级（`w_g=0`）  
- L2：向量降级（BM25+规则+最近事实缓存）  
- L3：保底（BM25+pinned facts）  
> 降级时不得绕过权限与安全策略。

## 7. 离线认知平面

### 7.1 睡眠巩固
按 user 分组聚合情景 → 归纳事实候选 → 冲突消解 → 版本写入 → 索引更新 → 产出质量报告。

### 7.2 主动遗忘
\[
ForgetScore = 0.7(1-TimeDecay) + 0.3(1-Imp)
\]
高 ForgetScore 先归档/失效，不直接物理删除。

### 7.3 反馈闭环
- 显式：点赞/纠错/删除/确认  
- 隐式：追问、引用、纠错缺失等信号  
- 上线前：离线评估 + 小流量验证

### 7.4 数据治理
- 分级：public/internal/sensitive  
- 保留：情景默认滚动保留，事实按证据有效性续期  
- 删除 SLA：逻辑删除与索引/备份清理有明确时限  
- 审计：操作人、trace、变更摘要完整留痕

---

## 8. 自进化控制平面（Phase 3）

### 8.1 进化范围
- 检索权重  
- 遗忘参数  
- 图结构阈值  
- 抽取 Prompt 模板

### 8.2 守护栏（必须）
- `quality >= baseline - ε`  
- `safety 不下降`  
- `P95 latency 涨幅 <= δ`  
满足后方可灰度。

### 8.3 停止条件
- 连续灰度失败自动熔断  
- 高峰时段冻结自动切换  
- 任一核心指标越阈值立即回滚  

### 8.4 验收清单
1. 离线质量不低于基线  
2. 灰度满足时延与安全  
3. 治理策略通过审计  
4. 降级演练通过  
5. 回滚脚本可快速恢复

---

## 9. 相对开源方案的优势总结

- **工程落地**：关系+向量+内存图，成本更可控  
- **治理能力**：证据链、审计、回滚、分级治理完整  
- **检索质量**：时间感知 + 图补充 + MMR + 动态配额  
- **演进能力**：有守护栏的自进化，不做无约束自动调参  
- **稳定性**：幂等一致性 + 分级降级 + 安全前置  

---

## 10. 分阶段实施路线

- **Phase 1（先稳）**  
  三层记忆 + BM25/向量检索 + 主动遗忘 + 幂等一致性 + 安全前置  
- **Phase 2（再优）**  
  睡眠巩固 + 反馈闭环 + 离线评测 + 数据治理闭环  
- **Phase 3（后智）**  
  轻量内存图 + 自进化控制 + 灰度/熔断/回滚全链路
